# Operating Model

## Commercial Flow

Proof360 turns founder context into commercial motion:

```text
Founder -> Gap -> Solution -> Margin -> Relationship
```

The product does not start by asking founders to fill out a form. It starts by reading what can be observed, then asking better questions.

## Roles

### Proof360

Finds the opportunity.

It identifies trust gaps, reconstructs a private Company Profile, and explains what is blocking investor, vendor, enterprise, insurance, or program readiness.

### CORPUS

Provides shared evidence.

CORPUS is not the same as Proof360 private memory. CORPUS is the broader permission-aware evidence layer for the estate.

### Ethiks360

Keeps the relationship.

Proof360 creates the conversation. Ethiks360 is where the trusted relationship can continue externally after the founder chooses that path.

## Memory Operating Rule

The Company Profile is reconstructed, not edited.

Founders contribute memory through:

- Conversation.
- Corrections.
- Cold-read sessions.
- Later evidence sources.

The system derives observations, maintains claims, and generates projections. It must not convert unknowns into fiction.

## Founder Experience Rule

Observation before recommendation.

Bad flow:

```text
You should buy X.
```

Good flow:

```text
Observed: no SOC 2 evidence, manual supplier records, MFA not enforced.
Possible next steps: ...
```

## Language Rule

Founder-facing language:

```text
Company Profile
```

Internal/export language:

```text
SPV candidate
SPV Passport
Ethiks360 projection
```

Do not expose SPV Passport language as the V1 entry experience.
